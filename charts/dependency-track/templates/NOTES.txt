The default admin credentials on first launch are `admin` / `admin`.
Dependency-Track forces a password change on first login.
Do this IMMEDIATELY, ideally before exposing the instance beyond `localhost`.

api-server pods run DT's init tasks on startup (database migrations,
default-object seeding, partition maintenance). A Postgres advisory lock serializes
concurrent runs; one pod migrates, the rest wait and then no-op.
If a pod stays `NotReady` for an unusually long time, inspect the api-server logs:

  kubectl --namespace {{ .Release.Namespace }} logs -l app.kubernetes.io/component=api-server -c api-server

{{- if and .Values.frontend.enabled (not .Values.frontend.apiBaseUrl) (not .Values.ingress.enabled) (not .Values.httpRoute.enabled) }}

WARNING: frontend.enabled=true but no `frontend.apiBaseUrl` is set and neither
`ingress` nor `httpRoute` is enabled. The UI will load but requests will fail to
reach the API server. Either:
  - enable Ingress / HTTPRoute (the chart will route /api -> api-server), or
  - set frontend.apiBaseUrl to a URL clients can reach (e.g. via port-forward).
{{- end }}
